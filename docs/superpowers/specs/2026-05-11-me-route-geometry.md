# ME-bridge ‚Äî route geometry CRUD (`me route` + `me waypoint`)

**Status:** approved (auto-approved via `/write-it`, 2026-05-11)
**Branch:** `feat/me-execution-bridge`
**Worktree:** `.worktrees/me-bridge`

## Goal

Add a verb surface to the ME-bridge for editing **route geometry only** ‚Äî adding, inserting, removing, listing, getting, and per-field-setting waypoints on a group's `route.points` array. The per-waypoint `task` field is read-preserved verbatim but explicitly **not** mutable here; that lives in follow-up sub-projects (per-waypoint main tasks, enroute tasks, commands).

Today the bridge has 80+ verbs (group, unit, zone, drawing, trigger, airbase, resources) but zero way to manipulate routes. `me_group_set_task.go` exists but sets only the group-level role string ("CAP", "CAS", "Escort", ‚Ä¶) ‚Äî explicitly *not* per-waypoint tasking, per its own source comment.

This is **sub-project #1** of a decomposed 4-part effort:

1. **Route geometry CRUD** ‚Äî this spec.
2. Per-waypoint main tasks (Orbit, AttackGroup, Bombing, EngageTargetsInZone, Land, Refueling, ‚Ä¶).
3. Enroute tasks + group-level mission task.
4. Waypoint commands (activate beacon, set ROE, switch waypoint, set frequency, EPLRS, smoke, transmit message, ‚Ä¶).

`framework/task.lua` (1082 lines, 25 builders) is the canonical task-table emission reference for #2 and #3.

## User value

After this spec lands, mission authors driving the bridge from CLI / agents / scripts can:

```bash
# inspect what's there
dcs-sms me route list   --group-name strike-1
dcs-sms me waypoint get --group-name strike-1 --index 2

# append / insert / remove waypoints
dcs-sms me waypoint add    --group-name strike-1 --north 240000 --east 380000 --alt 6000 --speed 220
dcs-sms me waypoint insert --group-name strike-1 --before 2 --north 250000 --east 390000
dcs-sms me waypoint remove --group-name strike-1 --index 3

# edit single waypoints
dcs-sms me waypoint set-pos    --group-name strike-1 --index 1 --north 245000 --east 385000
dcs-sms me waypoint set-alt    --group-name strike-1 --index 1 --alt 7500 --alt-type BARO
dcs-sms me waypoint set-speed  --group-name strike-1 --index 1 --speed 250
dcs-sms me waypoint set-type   --group-name strike-1 --index 4 --type "Land"

# wipe the route for a ground/ship group
dcs-sms me route clear --group-name convoy
```

This is the geometry foundation. Sub-projects #2‚Äì#4 will bolt task assignment onto the same waypoint identifiers.

## Constraints

- **Lua 5.1.** ME runs under Lua 5.1; the verbs.lua additions must too.
- **No throws.** Every verb returns `{ok = true, ‚Ä¶}` on success or `{ok = false, error = "<message>"}` on failure. Matches the existing failure mode (log + nil, never abort).
- **Drop-in additive.** No changes to existing verb signatures, no changes to dispatch contract, no changes to `runMeVerb` shape.
- **0-based wire / 1-based Lua.** CLI flags and JSON output use 0-based indices (matches what the ME UI displays). Lua internals work in their native 1-based `ipairs` world. `find_waypoint` translates at the boundary.
- **Task-field preservation invariant.** Every write verb must round-trip the per-waypoint `task` field untouched. Setters mutate only their target field. `inherit_waypoint` always emits an empty `{ id = 'ComboTask', params = { tasks = {} } }` ‚Äî never copies the source's task. `route_clear` and `waypoint_remove` drop tasks with the WPs they drop (correct).
- **Refresh after mutation.** Every write verb calls `refresh_group_view(g)` to re-render the F10 view. Matches the existing setter convention (`unit_set_pos`, `group_set_pos`, etc.).
- **Reuse existing helpers.** `find_group_in_mission(by_name, by_id)` at verbs.lua:416 returns `(group, country, side, category)`. `refresh_group_view(g)` at line 109 handles map-objects refresh.

## Decisions

### D1 ‚Äî One verb per field (matches existing `me unit set-*` convention)

Eleven per-field setters rather than a single bulk `waypoint update`. Matches the precedent set by `me unit set-pos/heading/alt/skill/livery/callsign/loadout/fuel/gun/chaff/flare/onboard_num` (13 setters for `unit`). Multi-field edits cost N bridge calls ‚Äî same trade-off as unit edits. Predictable for users moving between noun groups.

### D2 ‚Äî 0-based indexing on the wire

The ME UI displays waypoints as "Waypoint 0, 1, 2, ‚Ä¶". The bridge mirrors that. Lua internals stay 1-based; `find_waypoint(by_name, by_id, wire_index)` does the `lua_idx = wire_idx + 1` translation once at the boundary.

### D3 ‚Äî `--group-name | --group-id` (not `--name | --id`) for parent group

Verbose group flag avoids collision with the per-waypoint `--name` setter flag for waypoint names. `--group-name X` and `--group-id N` are mutually exclusive, exactly one required. Same shape on every route/waypoint verb.

### D4 ‚Äî Inheritance source on `add`/`insert`

On `add`: source = the last waypoint in the route (or category defaults if route is empty).
On `insert --before N`: source = the waypoint at index `N-1` (the one immediately before the insertion point). For `--before 0`, source = the waypoint currently at index 0 (the one being shifted up). If route is empty, source = nil, use category defaults.

Inherited fields: `alt`, `alt_type`, `speed`, `type`, `action`, `speed_locked`, `ETA_locked`, `formation_template`.
Never inherited: `name` (always empty unless `--name` passed), `ETA` (always `0` unless `--eta` passed), `task` (always empty ComboTask).
Always required: `--north`, `--east`.

### D5 ‚Äî Per-category defaults table

Used when adding/inserting into an empty route. Sourced from existing `group_create_*` defaults in verbs.lua to stay consistent:

| Category | alt | alt_type | speed (m/s) | type | action |
|---|---|---|---|---|---|
| `plane` | 8000 | BARO | 220 | Turning Point | Turning Point |
| `helicopter` | 500 | BARO | 50 | Turning Point | Turning Point |
| `vehicle` | 0 | BARO | 8 | Turning Point | Off Road |
| `ship` | 0 | BARO | 5 | Turning Point | Turning Point |
| `train` | 0 | BARO | 14 | Turning Point | On Road |
| `static` | (treated as vehicle for defaults; statics ignore routes anyway) |

`speed_locked = true`, `ETA_locked = true`, `formation_template = ""` for all categories.

### D6 ‚Äî Air groups must keep ‚â•1 waypoint

`waypoint remove` refuses for `plane`/`helicopter` if it'd leave 0 WPs. `route clear` refuses for `plane`/`helicopter` entirely (no escape hatch). Ground/ship/train/static can have empty routes (DCS treats as "stay put").

Error message: `"cannot remove last waypoint on air group '<name>'; use waypoint set-pos to reposition"` (for `remove`), `"cannot clear route on air group '<name>'; use waypoint set-pos to reposition"` (for `route clear`).

### D7 ‚Äî Enum validation against framework strings

`--type`, `--action`, and `--alt-type` validated as strict membership against the framework enums (`sms.waypoint.TYPE`, `sms.waypoint.ACTION`, `sms.alt_type` per `framework/waypoint.lua` and `framework/alt_type.lua`). Validation is duplicated in Lua-side tables (`WAYPOINT_TYPES`, `WAYPOINT_ACTIONS`, `ALT_TYPES`) ‚Äî the bridge runs inside ME, the framework runs inside the mission environment; they don't share runtime state. The duplicated tables are the seven type strings, eleven action strings, and two alt-type strings already catalogued in spec `2026-05-01-sms-skill-waypoint-enums.md`.

Unknown values ‚Üí `{ok = false, error = "unknown waypoint type '<X>'"}` (or action / alt_type). No "smart fallback" to a default ‚Äî bad input fails loud.

### D8 ‚Äî Speed in m/s, no convenience unit

DCS native, matches existing `group_create_plane` and friends. No `--speed-kts` shorthand (YAGNI; users can `awk` if needed).

### D9 ‚Äî Position as `--north m / --east m`

Bridge convention. Internally stored as `wp.x = north, wp.y = east` (.miz axis convention, where `y` means east-west on the ground plane ‚Äî verbs.lua header documents the trap at length).

### D10 ‚Äî All verbs added to `verbs.lua`, not a new file

Adds ~18 verbs + 4 helpers + 4 constant tables to an already-5661-line file (becomes ~7000+ lines). Splitting is the right long-term move but coupling it to new-feature work bloats the diff. Tracked separately in issue #54.

Verbs land in a contiguous block at the end of the existing setter section, prefixed by a section header:

```lua
-- ============================================================
-- Route / waypoint geometry verbs
-- ============================================================
```

### D11 ‚Äî Lua-side unit tests via new `mock_me_mission.lua`

This is the first Lua test suite that exercises `verbs.lua` directly (the other ~80 verbs have no Lua-side coverage; backfill tracked in issue #55). The mock module provides:

- `Mission.mission` ‚Äî synthetic mission table with coalition / country / category structure
- `Mission.update_group_map_objects(g)` ‚Äî no-op (records calls for assertion if needed)
- `Mission.create_group_map_objects(g)` ‚Äî no-op

The mock is registered via `package.preload['me_mission'] = ...` at the top of the test file so `require('me_mission')` inside verbs.lua picks it up.

No live bridge dispatch in Go tests ‚Äî that's the Lua layer's job. The Lua-arg-template check uses a captured fake `runMeVerb` to verify the templated string matches expectations.

### D15 ‚Äî Static groups handled as "ground for defaults, no-op at runtime"

Statics technically can have a `route.points` array in the .miz format but DCS ignores routes on statics at mission load. The bridge allows the verbs to run against statics (don't second-guess the user; future-task-scoped tooling may want them). Defaults table uses ground values. Min-WP rule: statics treated as ground (no enforcement). Documentation note in the spec; no explicit guard.

## Failure model

Standard error shape from Lua side: `{ok = false, error = "<message>"}`. Go side propagates as exit code 3 + JSON to stdout. Flag-parse errors ‚Üí exit code 2.

Per-condition error matrix:

| Condition | Error message |
|---|---|
| `args` not a table | `"<verb> requires args (table)"` |
| Neither / both of `--group-name` / `--group-id` | `"<verb> requires exactly one of args.name or args.id"` |
| Group not found | `"group not found: <identifier>"` |
| `--index` missing on a verb that needs it | `"<verb> requires args.index (integer ‚â• 0)"` |
| `--index` out of range | `"waypoint index <N> out of range (route has <K> points)"` |
| `--index` negative or non-integer | `"<verb> requires args.index (integer ‚â• 0)"` |
| `--before` out of range on insert | `"insert index <N> out of range (route has <K> points; legal range 0..<K>)"` ‚Äî note `--before K` is the append-equivalent and is legal |
| Required numeric field missing/bad type | `"<verb> requires args.<field> (number)"` |
| `--type` not in `sms.waypoint.TYPE` | `"unknown waypoint type '<X>'"` |
| `--action` not in `sms.waypoint.ACTION` | `"unknown waypoint action '<X>'"` |
| `--alt-type` not BARO/RADIO | `"alt_type must be 'BARO' or 'RADIO'"` |
| `--speed` ‚â§ 0 | `"speed must be > 0"` |
| `--eta` < 0 | `"eta must be ‚â• 0"` |
| `--alt` < 0 | `"alt must be ‚â• 0"` |
| `remove` would leave air group with 0 WPs | `"cannot remove last waypoint on air group '<name>'; use waypoint set-pos to reposition"` |
| `route clear` on air group | `"cannot clear route on air group '<name>'; use waypoint set-pos to reposition"` |
| `--locked` not bool-coercible | `"locked must be true or false"` |

## Testing

**Three layers:**

### 1. Go-side flag-parsing tests

One `*_test.go` per Go verb file. Table-driven, covers: missing required flag, mutex violation, neither identifier, bad type, happy-path Lua-template format. Captured fake `runMeVerb` checks template shape without hitting the bridge.

### 2. Lua-side unit tests (new `test_verbs_route.lua`)

Uses `mock_me_mission.lua`. Test matrix:

- `route_list`: empty route, multi-WP route, group not found, both --name and --id, neither
- `route_get`: full round-trip preserves `task` on every WP; preserves `routeRelativeTOT`
- `route_clear`: air group ‚Üí error; ground/ship/static ‚Üí success, returns `points_removed` count
- `waypoint_add`: empty route ‚Üí category defaults applied; non-empty ‚Üí inherits from last WP; explicit overrides win; `task` always empty ComboTask regardless of source WP's task; per-category defaults for plane/helo/vehicle/ship/train (when route empty)
- `waypoint_insert`: --before 0, --before middle, --before K (append-equivalent); index shifting verified by checking subsequent WPs; inheritance source = WP at `N-1` (or 0 for --before 0)
- `waypoint_remove`: middle, last; air group with multiple WPs ‚Üí success, with 1 WP ‚Üí error; ground with 1 WP ‚Üí success; out-of-range index ‚Üí error
- `waypoint_get`: happy path; out-of-range; missing index; group not found
- Per-field setters: one happy-path test per setter + one bad-arg test per setter (out-of-range index, type mismatch, enum rejection)
- **Task-preservation invariant**: for every write verb, synthesize a route where each WP has a non-empty `task.params.tasks`; run the verb; assert every non-touched WP's task table is identical (deep-equal via the existing `strip_back_refs` or a test helper)
- **Refresh invariant**: every write verb invokes `Mission.update_group_map_objects(g)` exactly once (mock records call count)

Register in `tools/me-mod/test/run-tests.ps1` `$tests` array.

### 3. Manual smoke (live ME)

Append to a `docs/me-bridge-smoke.md` checklist (create if absent):

- Open a mission in the ME
- `me route list` ‚Üí matches what ME shows
- `me waypoint add` ‚Üí new WP appears on F10 view
- `me waypoint set-pos` ‚Üí WP moves visually
